# 宝藏共鸣台 (Image Resonance) 说明文档

## 一、项目概述

**宝藏共鸣台**是一款 **图片素材管理 + 无限画布场景拼贴** 生产力工具，采用前后端分离架构。包含两大核心模块：

- **素材库**：管理分组 → 人物 → 图片，支持感知哈希去重、去白底、标签分类、批量操作
- **共鸣界面**：无限画布场景编辑器，支持多图层拖拽/缩放/旋转、编组、撤销重做、对齐吸附、导出 PNG/JPG

数据通过 MySQL + 文件系统存储于服务端，支持多人共享、完整备份导出/导入。

## 四、数据库设计

主数据存储在 MySQL 中，通过 Drizzle ORM 访问。IndexedDB（Dexie.js）仅用于备份导出/导入的本地缓存。

### MySQL 表结构（7 张表）

| 表名 | 关键字段 | 说明 |
|------|---------|------|
| `groups` | id, name, description, resonance_image_paths(JSON), sort_order | 人物分组 |
| `characters` | id, name, description, custom_fields(JSON), avatar_path, sort_order | 人物档案 |
| `character_groups` | id, character_id(FK), group_id(FK), unique(character_id, group_id) | 人物↔分组 多对多 |
| `images` | id, character_id(FK), original_path, processed_path, thumbnail_path, hash, tags(JSON) | 图片素材 |
| `tags` | id, name, color | 自定义标签 |
| `scenes` | id, name, background_color, background_image_path, layers(JSON), groups_json(JSON) | 画布场景 |
| `scene_templates` | id, name, scene_data(JSON), thumbnail_path | 场景模板 |

### 图片存储

图片以文件形式存储在 `server/uploads/` 下，MySQL 存储文件路径。图片通过 `/api/files/images/{id}/{variant}` 端点以 HTTP URL 形式提供（variant: `original` / `processed` / `thumbnail`）。

### IndexedDB（Dexie.js，仅用于备份）

备份导出/导入时使用 IndexedDB 作为本地缓存，表结构与 MySQL 一致。备份格式为 JSON（v2/v3 兼容），支持流式处理大文件。

## 六、核心算法说明

### 6.1 感知哈希去重 (pHash)

```
流程：图片 → 缩放至 16×16 → 转灰度 → 计算像素均值
     → 逐像素二值化（> 均值 = 1, ≤ 均值 = 0）→ 生成 64 位十六进制哈希
相似度：汉明距离（hamming distance）对比两个哈希值
```
- 实现位置：`src/utils/imageProcessing.ts` → `computePerceptualHash()` / `hammingDistance()`
- 上传时自动检测所有已有图片的哈希值，发现重复弹出提示

### 6.2 去白底算法（边缘连通 BFS）

```
1. 从图片四边出发，找到所有"浅色像素"（RGB 均 > 230 且通道间最大差异 < 15）
2. 使用 BFS 从边缘向内扩散，标记所有与边缘连通的浅色像素
3. 将被标记像素的 Alpha 通道设为 0（透明）
4. 内部不连通的浅色区域（如牙齿、白衣）不受影响，保持原样
```
- 实现位置：`src/utils/imageProcessing.ts` → `removeWhiteBackground()`
- 安全保护：60 秒超时，拒绝超过 6400 万像素的图片
- 上传时可选是否执行去白底

### 6.3 场景导出

```
离屏 Canvas 2x 分辨率渲染：
1. 遍历所有可见图层，计算导出范围（可视区域 / 全部图层）
2. 按 zIndex 排序依次绘制：背景色 → 背景图 → 各图层（含旋转/透明度/缩放变换）
3. 输出 PNG 或 JPEG Blob → 触发浏览器下载
```
- 实现位置：`src/utils/exportUtils.ts` → `exportScene()`

### 6.4 对齐吸附

```
图层拖拽时实时计算：
- 与被拖拽图层边缘/中心对齐的其他图层（垂直 + 水平方向）
- 阈值 5px 内自动吸附，显示辅助线
```
- 实现位置：`src/utils/alignment.ts` → `getAlignmentGuides()`

## 十、使用流程

### 素材库操作流程

1. 创建**分组**（如"草帽团"、"海军"等），支持拖拽排序
2. 在分组下创建**人物**（人物可归属多个分组）
3. 为人物编辑**属性档案**：简介、职位、果实能力、霸气、身高、生日、自定义字段
4. 为人物设置**头像**（从已上传图片中选择或裁剪）
5. 选中人物，点击**上传图片**，选择一张或多张图片
6. 上传时自动进行**哈希查重**检测，可选是否**去白底**
7. 为图片添加**标签**（可批量操作），支持按标签颜色分类
8. 使用**搜索框**和**标签筛选**快速定位素材
9. 图片网格支持**拖拽排序**、**拖拽到画布**

### 共鸣界面操作流程

1. 切换到**共鸣界面**视图
2. 在左侧人物库面板找到人物，点击图片缩略图添加到画布
3. 在画布上**拖拽移动**、**边角缩放**、**顶部手柄旋转**图层
4. 拖拽时自动显示**对齐辅助线**（边缘 + 中心吸附，5px 阈值）
5. 使用**属性面板**精确调整位置/尺寸/旋转/透明度
6. **右键菜单**或**快捷键**进行复制/粘贴/删除/编组/层级调整/锁定
7. 通过**图层列表面板**管理图层树（排序/可见性/锁定/编组展开）
8. 使用 `Ctrl+Z/Y` **撤销/重做**（最多 50 步历史）
9. 点击顶部**导出**按钮将场景输出为 PNG/JPG（可选可视范围/全部、背景色）
10. 点击**保存**按钮持久化场景数据到 IndexedDB
11. 通过**场景管理**创建多场景、复制场景、从模板创建、存为模板

## 十一、数据安全

- **服务端存储**：所有数据存储在 MySQL + 文件系统，支持多设备访问
- **备份导出**：通过备份功能导出 ZIP 文件（含所有图片 + JSON 数据，支持流式处理大文件）
- **备份导入**：支持导入备份 ZIP 恢复数据（增量导入，不覆盖已有数据）
- **版本兼容**：导入时自动兼容 v1/v2 旧格式数据（自动迁移 character.groupId → characterGroups 联结表）
- **二次确认**：所有删除操作均需确认，防止误操作
- **级联删除**：删除分组 → 解除关联 → 删除无归属人物 → 删除其全部图片
